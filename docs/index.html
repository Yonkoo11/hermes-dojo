Your Agent, Getting Better While You Sleep

Hermes Dojo reads your agent's session logs, finds recurring failures, and creates skills to fix them. Self-improvement that actually works.

The Problem
Your agent forgets everything

Every session starts from zero. Corrections vanish. The same failures repeat endlessly.

Same Mistakes, Every Day

You correct a wrong git branch. Next session, same error. The agent has no memory of what went wrong.

No Feedback Signal

86 skills installed, but which ones work? Which silently fail? Nobody tracks per-skill success rates.

Evolution Unused

GEPA self-evolution exists. But without knowing what to evolve, it sits idle while your agent stagnates.

How It Works
Five steps to a smarter agent

Dojo closes the feedback loop. Measure, identify, fix, evolve, report.

01

Monitor

Reads session logs from state.db. Identifies tool errors, retry loops, user corrections, and explicit complaints. Computes per-skill success rates.

02

Analyze

Categorizes root causes and ranks weaknesses by impact. "web_extract fails 100% of the time. Root cause: no retry logic for timeouts."

03

Fix

Patches failing skills with targeted error handling. Creates new skills for missing capabilities. Injects environment variables and configuration.

04

Evolve

Runs GEPA self-evolution on weak skills. DSPy-based prompt optimization using real failure data as signal, not random mutation.

05

Report

Generates improvement reports for CLI or Telegram. Sparklines, deltas, and actionable summaries delivered to your morning inbox.
